The premise

Make secrecy look like history.

A writer types ordinary English. The browser encrypts it with a shared key, converts the encrypted data into Voynich-like words, and sends only the mysterious text. The recipient enters the same key to recover the original message.

“Anyone without the key sees a convincing page of Voynichese.”
01 — ENCRYPT

Protect the meaning

AES-256-GCM provides real authenticated encryption. Identical messages produce different output.

02 — DISGUISE

Wear the manuscript

Encrypted bits become tokens assembled from EVA-inspired Voynich word patterns.

03 — REVEAL

Share one secret

The correct passphrase reverses the process. A wrong key reveals nothing useful.

How it works

Real encryption, theatrical encoding.

LayerPurposeCurrent prototype
Key derivationTurns the shared phrase into a 256-bit keyPBKDF2 with a random 16-byte salt
EncryptionProtects and authenticates the messageAES-GCM with a random 12-byte nonce
EnvelopeCarries version, salt, nonce, and ciphertextBinary VM1 packet
Voynich encodingTurns binary into manuscript-like words4,096 unique tokens; 12 bits per word
PresentationMakes output resemble a manuscript passageEmbedded original “Voynich Messenger” glyph font

The visible dictionary is not the security. Even if an observer converts every Voynich token back into bytes, those bytes remain authenticated ciphertext. The shared key is what unlocks the meaning.

Authentic mode vs. generative mode

AUTHENTIC

Use only word forms documented in manuscript transcriptions. More historically faithful, but dependent on a cleaned and licensed dataset.

GENERATIVE

Construct reversible words from EVA-inspired components. Larger vocabulary and simpler encoding. This demo uses this mode; most output words are invented and are not claimed to occur in the manuscript.

STORY MODE

Let token families suggest imagined concepts—water, source, drain, body, plant—so messages resemble narrative Voynich passages.
